# Single-thread and Non-blocking event loop concurrency model

Concurrency Modal?

Oru periya task run aana ena aagum?

  • Remote server la irundhu oru data featch pani edukurom idhu single thread ah block panidum
  • Ana namaku apdi aaga kudadhu
  • Namaku non-blocking ah code run aaganum → Event Loop

Event Loop

Event Loop oru periya task eduthu adha background la execute panum adhu run aana aprm marupidium adha main thread la ye potudum complete aana aprm

Concurrency Modal?

What if there is a long-running task?

  • Fetching data from a remote server well, it sounds like that would block the single thread
  • But of course we don't want that
  • We want is so-called non-blocking behaviour → Event Loop

Event Loop

The event loop takes long-running tasks executes them in the background and then puts them back in the main thread once they are finished.
